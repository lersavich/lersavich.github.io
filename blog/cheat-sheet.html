Ruby Enumerable Method Cheat Sheet

Ruby is an object-oriented programming language for computers. As part of its makeup, a certain amount of functions have already been defined for the user, and can be called with a simple mention of their name. Here below is a list of all such "enumerable" methods, which are functions that specifically take a set of data and do something to the data. (Note that a "collection" as referred to on this page can mean either an array or a hash.)

To see the official Ruby Docs write-up for any particular method, click the method's name.

#all?

Do all the elements in a collection meet a specific criteria?

#any?

Do any of the elements in a collection meet a specific criteria?

#chunk

Run a block of code on a collection then return the values as "chunks" of sub-arrays*

*Only groups consecutive elements into sub-arrays

#collect

Run a block of code on an array, then collect the results into a new array*

*Same thing as #map

#collect_concat

Run a block on an array, and flatten the results as one big array itself*

*Same thing as #flat_map

#count

Count the number of items in a collection*

*Similar to both #size and #length, except in #count you can specify a really specific thing to be counted

#cycle

Cycle through a block of code a specified number of times

#detect

Find the first instance in a collection of something that matches a block of code*

*Same thing as #find

#drop

Drop a number of items from the beginning of a collection, as defined by the user

#drop_while

Drop items from a collection, but only if the item meets a certain criteria*

*Only does this until coming across the first item that returns false

*Opposite is #take_while

#each_cons(n)

For each iteration of a loop, perform a block of code on “n” consecutive elements in a row, before moving on to the next iteration

*Note that this goes through the iteration one element at a time like normal, just that it performs a block of code on multiple items in that collection each time

#each_entry

Convert a yield into a collection

#each_slice

Perform a block of code on a collection, but doing it by a certain amount of items all at once, then returning those chunks as separate collections

#each_with_index

Perform a block of code on a collection, but also while calling up each index number at the same time

#each_with_object

Perform a block of code on a collection, but with an arbitrary object argument added

#entries

Converts an object into an array

*Same exact thing as #to_a

#find

Find the first instance in a collection of something that matches a block of code

*Same thing as #detect

#find_all

Perform a block of code on a collection, and output all items that return true*

*Same exact thing as #select

#find_index

Perform a block of code on a collection, and return the index number of the first value that is true

#first

Return the first value of a collection, or the first “n” values as defined by the user

#flat_map

Run a block on an array, and flatten the results as one big array itself*

*Same thing as #collect_concat

#grep

Iterate over a collection and see if any parts of each value match a given regular expression*

*If a block of code is added, each result that returns true will be passed through the block

#group_by

Run a block of code, then group the results into a hash, with each hash’s key being the number of that iteration

#include?

Does the collection include the value being defined?

#inject

Run a block of code on a collection, then add the results to a new variable with each iteration*

*The initial value of that new variable can be pre-defined, or else its initial value will be the value after the first iteration

*Same thing as #reduce

#lazy

Return a “lazy” value of certain enumerables, by combining their results with a certain yield

#map

Run a block of code on an array, then collect the results into a new array*

*Same thing as #collect

#max

Return the item in a collection with the maximum value.

#max_by

Return the item in a collection with the maximum value, as determined in a block of code.

#member?

Does anything in a collection match the object the user is mentioning?

#min

Return the item in a collection with the minimum value.

#min_by

Return the item in a collection with the minimum value, as determined in a block of code.

#minmax

Return the two items in a collection with the minimum and maximum value.

#minmax_by

Return the two items in a collection with the minimum and maximum value, as determined in a block of code.

#none?

Is it true than none of the items in a collection can pass a certain block of code?

#one?

Is it true than one and only one item in a collection can pass a certain block of code?

#partition

Return two arrays, the first containing the items in a collection that tested "true" for a block of code, the second containing everything left.

#reduce

Combines all elements of enum by applying a binary operation, specified by a block or a symbol that names a method or operator.

#reject

Returns an array for all elements of enum for which the given block returns false. If no block is given, an Enumerator is returned instead.

#reverse_each

Builds a temporary array and traverses that array in reverse order. If no block is given, an enumerator is returned instead.

#select

Returns an array containing all elements of enum for which the given block returns a true value.

#slice_before

Creates an enumerator for each chunked elements. The beginnings of chunks are defined by pattern and the block.

#sort

Breaks a collection into groups based on some given criteria. It returns these groups as a hash where each key represents each group and the value is an array containing the member of each group.

#sort_by

Sorts enum using a set of keys generated by mapping the values in enum through the given block.

#zip

Takes one element from enum and merges corresponding elements from each argument.
